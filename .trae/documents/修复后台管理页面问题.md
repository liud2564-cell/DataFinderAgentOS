# 修复后台管理页面问题

## 问题汇总
1. 后台标题"智能瞭望与问数"换行显示，CSS样式需要调整
2. 用户管理、角色管理、功能管理、菜单管理页面需要统一为后台主界面布局（上logo+菜单区，下主界面区，左菜单右工作区）
3. 用户管理API报错：`users`表缺少`status`列
4. 菜单管理页面报错：`dict`对象访问属性错误

## 修复方案

### 步骤1：修复数据库表结构
**文件**: `app/models/db.py`
- 在`init_db()`函数中为`users`表添加`status`字段（默认值为1）
- 修改SQL语句：`status INTEGER DEFAULT 1`

### 步骤2：修复后台标题换行问题
**文件**: `app/templates/admin/index.html`
- 在`.layui-layout-admin .layui-logo`样式中添加`white-space: nowrap`防止换行
- 或缩短标题文字

### 步骤3：重构各管理页面为统一布局
**需要修改的文件**:
- `app/templates/admin/user_management.html`
- `app/templates/admin/role_management.html`
- `app/templates/admin/function_management.html`
- `app/templates/admin/menu_management.html`

**改造方案**:
每个页面需要：
1. 保留现有的管理功能区域（搜索栏、表格、弹窗等）
2. 添加后台管理布局结构：
   - 顶部导航栏（logo + 用户菜单）
   - 左侧菜单（用户管理、角色管理、功能管理、菜单管理）
   - 右侧工作区（原有内容）
3. 左侧菜单中当前选中的模块高亮显示
4. 各模块之间可以通过菜单切换

### 步骤4：修复菜单管理页面dict属性访问错误
**文件**: `app/templates/admin/menu_management.html`
- 第46行：`role.id` 改为 `role['id']`
- 因为传递的是`dict`列表，需要使用字典访问方式

## 实施顺序
1. 先修复数据库表结构（步骤1）
2. 修复标题换行问题（步骤2）
3. 修复菜单管理页面dict访问错误（步骤4）
4. 最后重构各管理页面布局（步骤3）

## 验证方法
1. 重启服务器后访问后台管理页面
2. 检查标题是否正常显示在一行
3. 访问各管理页面，确认布局统一且可以切换
4. 访问用户管理API，确认不再报数据库错误
5. 访问菜单管理页面，确认不再报dict属性错误
