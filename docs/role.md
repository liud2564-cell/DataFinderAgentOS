# DataFinderAgentOS 系统角色说明

## 系统背景
- **系统名称**：某政务智能瞭望与智能问数系统
- **系统类型**：Web B/S 结构的数据智能采集和数据智能问数系统
- **架构划分**：用户侧-前台 + 管理侧-后台
- **用户侧定位**：类 ChatGPT 的智能应用，用户通过 Chat 模式与数据直接对话，实现智能问数，通过模型直接获得数据统计、分析、生成图形图像报表
- **管理侧定位**：后台管理系统，负责用户、权限、数据采集、模型引擎、数字员工、大屏展示等管理功能

## 文档定位
- 本文件定义系统的用户角色体系、权限边界和各角色的功能范围。
- 角色体系基于 RBAC（Role-Based Access Control）模型，与 `roles` 表和 `role_functions` 表对应。

---

## 角色体系概览

| 角色编码 | 角色名称 | 类型 | 状态 |
|---------|---------|------|------|
| `admin` | 超级管理员 | 系统预置 | 固定启用 |
| `user` | 普通用户 | 系统预置 | 默认启用 |
| （自定义） | 自定义角色 | 用户创建 | 可配 |

---

## 预置角色详情

### 超级管理员（admin）
- **编码**: `admin`
- **描述**: 系统最高权限角色，拥有所有功能模块的完整操作权限
- **保护机制**: 不可删除、不可修改、不可禁用
- **权限范围**:
  - 后台管理侧：所有模块（用户管理/功能管理/菜单管理/角色管理/瞭望管理/数据管理/采集管理/模型引擎/数字员工/数智大屏/舆情大屏）
  - 用户侧前台：所有功能（登录/注册/对话/数字员工/@天气/@新闻/@音乐/@电影/@yummy/报表/历史/导出）
- **标识**: 后台用户管理中显示 `[管理员]` 标识，删除按钮自动隐藏

### 普通用户（user）
- **编码**: `user`
- **描述**: 系统普通用户，默认注册后的角色
- **权限范围**:
  - 用户侧前台：登录、对话、数字员工调用、@技能命令、报表查看、历史记录、报告导出
  - 后台管理侧：无管理权限（不能访问 `/admin/` 路由）
- **可配置**: 可通过角色-功能分配扩展权限，允许授予部分后台管理权限

## 自定义角色
- 管理员可通过"角色管理"模块创建自定义角色
- 创建时可从完整功能树中选择分配给该角色的功能
- 分配功能后系统自动生成对应的菜单条目
- 支持启用/禁用状态切换
- 支持删除（管理员保护除外）
- 自定义角色可混合分配后台管理和用户侧功能，实现精细化权限控制

---

## 权限判定流程

### 后台访问控制
```
请求 → BaseHandler.get_current_user()
  → 验证 secure_cookie 中的用户名
  → 查询用户信息（含 role_id）
  → 查询 role_functions 表获取功能权限列表
  → 查询 menus 表构建菜单树
  → 渲染页面时 user_info 包含 menu_tree 和 role_id
```

### 后台路由保护
```
@tornado.web.authenticated 装饰器
  → secure_cookie 验证
  → 未登录 → 重定向到 /admin/
```

### 前台访问控制
```
请求 → LoginHandler/IndexHandler
  → 验证 secure_cookie 中的用户名
  → 未登录 → 渲染登录页
  → 已登录 → 渲染首页
```

---

## 用户管理规范
1. 每个用户必须分配一个角色（`role_id` 不可为空）
2. 用户注册时默认分配 `user` 角色（`role_id=2`）
3. 用户在后台创建时可选择任意角色
4. 删除用户时不会删除关联的角色
5. 修改用户角色后，用户的权限在下次请求时生效

---

## 安全约束
1. 任何用户不能修改/删除 `admin` 角色
2. `admin` 角色的用户不能被修改用户名和删除
3. 角色-功能分配变更时，菜单自动同步更新
4. 前端菜单树由服务端动态生成，前端不存储权限数据
5. 未授权用户不能通过猜测 URL 访问后台页面
6. 前后台使用独立的登录入口（`/` 对应用户侧，`/admin/` 对应管理侧）
